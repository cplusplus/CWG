2861. dynamic_cast on bad pointer value

Section: 7.6.1.7  [expr.dynamic.cast]     Status: open     Submitter: Jim X     Date: 2024-02-06

(From submission #497.)

Base-to-derived casts and cross-casts need to inspect the vtable of a polymorphic type. However, this is not defined as an "access" and there is no provision for undefined behavior analoguous to 7.2.1 [basic.lval] paragraph 11.

Possible resolution:

Add a new paragraph after 7.6.1.7 [expr.dynamic.cast] paragraph 6 as follows:

If v is a null pointer value, the result is a null pointer value.

If v has type "pointer to cv U" and the dynamic type of the object pointed to by v is not similar (7.3.6 [conv.qual]) to U, the behavior is undefined. If v is an lvalue and the dynamic type of the object referred to by v is not similar to the type of v, the behavior is undefined.
